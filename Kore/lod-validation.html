3. LOD & PALOD Field Rules (Quick View)

These are the key checks this tool enforces. Use them as a quick visual guide when reviewing a file by eye, or to explain to others why a row failed validation.

Core Identifiers & Order Basics

  • Group Code – All caps (for example, SAMS).
  • Customer Number – Must be a valid ship-to; cannot end in 00.
  • Item Number – Required, no extra restrictions in this tool.
  • Quantity – Required and numeric; must be greater than zero.
  • Order Type – In caps and limited to GO, GR, PA, or DO.
  • Allocation Rule – All LODs must be entered as O.

Customer PO & PALOD Logic

  • Customer PO (LOD) – Required, max 20 characters and unique per address.
  • Address uniqueness – If the same PO appears with a different address block, it is flagged.
  • PA-routed LOD rows – When Order Type is PA, the Customer PO must start with 1NAPR.
  • Customer PO (PALOD) – Every row must have a Customer PO starting with 1NAPR.

Address & Contact Details

  • Name – Customer name / ship-to name.
  • Ship-to Address 1 – Contact name / title / company; not the street line. Max 35 characters.
  • Ship-to Address 2 – Street address or PO Box. Do not combine multiple addresses on one line.
  • Ship-to Address 3 – Suite / building numbers or phone numbers longer than 10 digits.
  • City – Required.
  • State – Two-character U.S. state code in caps (for example, NY).
  • Zip – Preserve leading zeros. Five or nine digits total once hyphens are removed.
  • Country – Two-character code in caps (for example, US, not USA).
  • Contact – 10-digit phone number with no spaces or dashes.

Dates, Shipping & Emails

  • Customer In Hands Date – Required; format MM.DD.YYYY (for example, 04.12.2013).
  • On or Before Indicator – Must be O (on) or B (before).
  • Must Ship Date – Optional; only used with specific DC instructions, format MM.DD.YYYY.
  • Address Over-Ride – Y or N; typically Y because addresses are pre-verified.
  • Service Level of Shipment – In caps (for example, BEST, NEXT, 2DAY, 3DAY, MAIL).
  • Carrier – Only use when requesting a specific carrier; typically left blank.
  • Ship From Warehouse – Should be blank for standard LOD processing.
  • Confirmation Email – Multiple addresses separated with ;. Watch for hidden spaces and ,com typos.
  • Ship Confirmation Email – Same rules as Confirmation Email.
  • Site – Required when using confirmation emails; determines the model site for the order.

Tip: Use this section as a “pre-flight checklist.” If a file constantly fails on the same column, you can quickly scan here, fix the pattern in Excel, then re-run validation until the metrics show zero issues.
