LOD Validation Tool

Strict, offline validation for LOD and PALOD files before NVISION ingestion.

1. Configuration & Upload

Reference: Applied Logic Rules

Core Identifiers

  • Group Code: Must be uppercase (e.g., SAMS).
  • Customer Number: Must be numeric; cannot end in 00.
  • Quantity: Must be a numeric whole number greater than 0.
  • Order Type: Strictly limited to GO, GR, PA, or DO.
  • Allocation Rule: Must be exactly O.

PA & PALOD Strict Logic

  • LOD Mode: If Order Type is PA, the PO number must begin with the 1NAPR prefix.
  • PALOD Mode: ALL rows require the PO to start with 1NAPR, regardless of Order Type.
  • Uniqueness: A PO number applied to a different address in the same file will be flagged. Maximum 20 characters.

Formatting Standards

  • State / Country: Exactly 2 uppercase letters (e.g., NY, US).
  • ZIP Code: 5 or 9 digits (hyphens are automatically stripped and validated).
  • Phone Number: Exactly 10 digits (no spaces or hyphens allowed).
  • Email Addresses: Cannot contain spaces. Must use .com (flags ,com typos). Multiple emails must be separated by a semicolon ;.
