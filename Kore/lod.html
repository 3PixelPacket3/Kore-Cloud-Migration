Reference: Applied Logic Rules

Core Identifiers

  • Group Code: All caps (e.g., SAMS).
  • Customer Number: Numeric; cannot end in 00.
  • Quantity: Numeric, > 0.
  • Order Type: GO, GR, PA, or DO.
  • Allocation Rule: Strictly O.

PA & PALOD Strict Logic

  • LOD: If Order Type is PA, PO must start with 1NAPR-.
  • PALOD: ALL rows require PO to start with 1NAPR-.
  • Uniqueness: Same PO applied to a different address will flag. Max 20 chars.

Address & Contact Formatting

  • Address 1: Contact/Company (Max 35 chars).
  • Address 2: Street (Do not combine PO Box & Street).
  • State: 2-char uppercase (NY).
  • ZIP: 4, 5, 8, or 9 digits (accommodating zero cut-off; hyphens removed).
  • Country: 2-char uppercase (US).
  • Phone: Exactly 10 digits (no spaces/hyphens).

Execution & Logistics

  • Dates: Either In Hands or Must Ship is required. Format MM.DD.YYYY.
  • Indicator: Must be O (on) or B (before).
  • Address Over-Ride: Y or N.
  • Service Level: Uppercase (NEXT, 2DAY).
  • Emails: No spaces. Use .com (not ,com). Delimit with ;.
