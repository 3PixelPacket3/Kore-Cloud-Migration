## Work Tracker – Deep Guide

The Work Tracker is the operational backbone of Kore.

It is designed to capture incoming work requests, tasks, and follow‑ups in a
single queue that supports prioritization, filtering, and lightweight reporting.

### What the Work Tracker Is
- A request intake queue
- A task management view
- A lightweight activity log
- A reporting data source

### Creating Work
Use **Add Request** to open the dedicated Work Request page.
Each request supports:
- Title and description
- Request type
- Account or category
- Priority and status
- Due date
- Notes (rich text)

### How Data Is Stored
- All work items are saved in browser storage (`localStorage`)
- No server or backend is required
- Work items can be exported to CSV at any time

### Best Practices
- Keep titles concise and descriptive
- Use request types and accounts consistently
- Update status instead of deleting items
- Export CSV periodically for archival or analysis
