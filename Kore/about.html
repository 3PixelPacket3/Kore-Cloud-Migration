About Kore

Kore is a local-first personal work and knowledge system. The name stands for KORE – Knowledge Organization & Retrieval Engine. It runs as an offline website inside a folder on your machine and opens in any modern browser.

1. What Kore Is

Kore is a local website: you open it by double-clicking index.html in the extracted Kore folder. There is no server or sign-in. Everything is designed to work completely offline.

The goal is to give you a single hub for:

  • Capturing how you do things (procedures, checklists, troubleshooting steps).
  • Standardizing those processes into clean, repeatable guides.
  • Retrieving them quickly using the Info Hub and Ask Kore.

Think of Kore as your personal operations manual that lives entirely on your machine.

2. Where Your Data Lives

Important: Kore’s content is stored in your browser’s localStorage. Clearing browser data or changing browsers will remove that content unless you have a backup.
  • Kore files (the app): the folder where you extracted the zip (HTML, CSS, JS, images).
  • Your content: pages, apps, docs, work items, and settings stored in the browser.
  • Your backups: JSON files you download using the backup tools in Kore.

3. Core Areas of Kore

Kore is organized into a few core areas:

  • Home: clock, date, time zone, hero image, and homepage message.
  • Info Hub: your knowledge layer – how-to guides and reference pages.
  • Ask Kore: an offline assistant that points you to the right page or control.
  • Settings: profile name, theme, homepage HTML, backup/restore tools.
  • Tools (Apps, Work, Docs):
    • Apps: an external Apps & Tools Hub of tiles for the sites and utilities you use most.
    • Work: a request-style Work Tracker queue with type, priority, status, and notes for each ticket.
    • Docs: a multi-file Documents Library for PDFs, Office files, CSVs, text, images, and cloud links.

Detailed “how-to” instructions for tools like Apps, Work, and Docs are best kept as Info Hub pages so you can version and refine them over time.

4. First-Time Setup

Initial configuration

  1. Open Home and confirm your time zone.
  2. Go to Settings and rename the profile from User to your name or role.
  3. Still in Settings, update the Homepage Message with your own HTML if desired.

Sanity check

  1. From Home, verify the clock and date show correctly for your time zone.
  2. Click Save Backup in the top-right to download your first backup file.
  3. Optionally, add a simple test page in the Info Hub to confirm saving works in this browser.

5. Building Guides in the Info Hub

The Info Hub is where your knowledge lives. Each page is a guide, checklist, or reference topic you want to be able to find again quickly.

Create a new guide

  1. Open Create a Page.
  2. Enter a clear Title and a short Summary (both support HTML).
  3. Use the main editor for your steps. You can switch between visual and HTML mode.
  4. Add tags (comma-separated) so pages can be filtered and found by Ask Kore.
  5. Attach supporting images and links as needed.
  6. Save as Draft while editing, and Publish when it is ready.

Recommended structure

  • Overview: what this guide covers and why it exists.
  • Prerequisites: access, tools, or files required.
  • Step-by-step: numbered steps with exact button/field names.
  • Verification: how to confirm it worked.
  • Troubleshooting: common failure modes and fixes.
  • Keywords: optional line with search terms to help Ask Kore match queries.

6. Using Ask Kore

Ask Kore is an offline assistant. It does not use ChatGPT or any external model. Instead, it matches your question against:

  • Kore’s built-in help content (this About page, Settings, Home, etc.).
  • Your Info Hub guides (titles, summaries, content, and tags).
  • Tool metadata (Apps, Work items, Docs names) when you search “Everything”.

To get the best results:

  • Use the same wording that appears in your guides (system names, task names, key nouns and verbs).
  • Use the Scope selector to narrow searches to Core Pages or Info Hub only.
  • When you create a new guide, repeat important phrases in the body and tags so Ask Kore can match them.

7. Backups, Restore & Moving Between Machines

Backups are how you protect your Kore content and move it between devices. This is also the practical way to "sync" a newer version of Kore (for example, a fresh zip with UI changes) with your existing work.

Create a backup

  1. Click Save Backup in the top-right navigation on any page, or
  2. Open Settings and select Download Backup.
  3. Store the backup .json file in a safe folder (for example, a Kore Backups directory).

Restore on another machine

  1. Copy the latest Kore folder (from the new zip) to the target machine.
  2. Open index.html to launch Kore, then go to Settings.
  3. In Backup & Restore, choose a Restore mode and Tools option:
  • Merge with current data: keep existing content and add everything from the backup.
  • Replace current data: overwrite current content with the backup's content.
  • Include Tools data (Apps, Work, Docs, Tasks): check this if you want those areas restored from the backup as well; uncheck to keep the current Tools content and only bring in Info Hub and core settings.

Finally, use Restore from File to select your backup.

For a "JanSan-style" master backup, keep a clean backup that contains your baseline guides and references. You can merge that into a fresh Kore folder on another machine so they receive the latest guides while keeping their own Tools data intact (by unchecking the Tools option).

8. Launching Kore Quickly (Shortcuts)

You can make Kore feel more like an "app" by creating shortcuts to index.html.

Desktop shortcut (Windows)

  1. Open File Explorer and navigate to the folder that contains index.html.
  2. Right-click index.html and choose Send to → Desktop (create shortcut).
  3. Rename the shortcut to something like Kore.
  4. Double-click that shortcut anytime to open Kore in your default browser.

Browser favorites

  1. Open index.html in your browser.
  2. Add it to your bookmarks or favorites bar.
  3. Optionally, pin the tab so Kore is one click away whenever the browser opens.
